# Lessons Learned
> Accumulated rules from corrections and mistakes. The agent reviews this file at the start of every session and updates it after every correction.

## How This File Works
- **After ANY correction from the user**, add a new entry below with the pattern and the rule to prevent it.
- **At session start**, read this file and apply all rules for the current project.
- **Iterate ruthlessly** — if a mistake recurs, strengthen the rule or add a more specific one.
- This is NOT the same as `reflection-log.md` (which is narrative post-task history). This file is **rules only** — short, actionable, enforceable.

## Format
```markdown
### [YYYY-MM-DD] Short description of the mistake
- **What happened:** One sentence.
- **Root cause:** Why it happened.
- **Rule:** The rule to prevent it. Start with "Always..." or "Never..."
```
